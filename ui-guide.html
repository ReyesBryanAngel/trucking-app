01 — Screens

All App Screens

Three navigable screens make up the current app: Trip List, Trip Detail, and Pre-Trip Inspection. The screenshots below were captured on Android Emulator (Medium Phone API 35).

Trip List

Daily overview summary + scrollable trip card list. Entry point of the app.

Trip Detail — Pending

Truck & driver info, cargo details, and the "Start Pre-Trip Inspection" action.

Pre-Trip Inspection

Checklist of 5 safety items. Button stays disabled until all items are checked.

Inspection — All Checked

Progress bar fills to 100% and the "Submit Inspection" button activates.

Inspection Passed Dialog

Alert confirms all items passed and trip status advances to "Ready to Start".

Trip Detail — Ready to Start

"Start Trip" and "Reset to Pending" actions are now available.

Trip Detail — In Progress

Trip start timestamp appears. Driver can mark delivered or report a delay.

Trip Detail — Delayed

Amber warning note box appears. Only "Reset to Pending" is available.

02 — Trip Flow

Status Lifecycle

Each trip moves through a fixed set of states. The "Reset to Pending" action is available on any status except Pending, allowing a driver to restart the flow.

From Status Action Next Status Notes
Pending Start Pre-Trip Inspection — (navigates to inspection screen) All 5 checklist items must be ticked
Pending Submit Inspection Ready to Start Sets inspectionCompleted: true
Ready to Start Start Trip In Progress Stamps startedAt timestamp
In Progress Mark as Delivered Delivered Requires confirmation alert. Stamps deliveredAt
In Progress Report Delay Delayed Requires confirmation alert
Any (except Pending) Reset to Pending Pending Clears startedAt, deliveredAt, resets inspectionCompleted

05 — Components

Shared Components

All reusable components live in src/components/.

TripCard

Renders a single trip row in the list: ID, status badge, plate chip, route visualiser, cargo, and ETA.

StatusBadge

Pill badge that maps a TripStatus to its background and text color via STATUS_COLORS.

status: TripStatus

PrimaryButton

Full-width action button with four color variants, a loading spinner state, and disabled styling.

labelvariantloadingdisabled

SummaryCard

Dark overview card at the top of the list showing Total, In Progress, Delivered, and Delayed counts.

ChecklistItem

UI ActionScreenHandlerContext Method
